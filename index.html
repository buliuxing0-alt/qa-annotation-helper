📌 第一步：描述视频场景
用一两句话描述视频内容（类型、主题、关键人物/品牌），这样生成的替换题才不会跑偏。
💡 批量修复：先填写上方的「视频场景描述」，然后粘贴多行题目数据。
格式：类型 | 提问时间 | 问题 | 答案 | 解析（用 | 或 tab 分隔）

📌 各类型出题核心结构

A 实时多模态感知

核心结构：声音提到的X + 画面显示的Y → 关联判断

示例：

  • 「主持人现在说的这个人，是否就是大屏照片里站在左边的那位？」
  • 「旁白刚才说的那个'偷东西的小偷'，是否就是当前画面中叼着食物的这只老鼠？」
B 历史回顾

核心结构：当前状态 vs 早前状态 → 对比回顾（跨度≥5分钟）

示例：

  • 「当前比分领先的一方，是否就是第一节结束时落后的那一方？」
  • 「截止当前，场上得分最高的球员是谁？」（进球时间 3:10，提问时间 62:02）
D 工具调用（50%，核心）

核心结构：画面中的事物 + 历史回顾 + 外部搜索 → 多层叠加

示例：

  • 「当前屏幕中坐在替补席最左侧的这位球员，是否有过国家队出场经历？」
  • 「主播现在带的这款产品所属公司，和第一个商品链接的售价差额是多少？」
  • 「这家公司与主播介绍的上一款产品所属公司之间是否存在合作关系？」
E 主动交互

核心结构：事件触发 → 直接复述（10分钟内触发）

示例：

  • 「下半场开场的时候立刻告诉我"下半场开始了"」→ 答「下半场开始了」
  • 「一旦乔丹投进三分了立刻提醒我」→ 答「乔丹投进三分了」
❌ 常见错误
  • A类：纯看画面或纯听声音就能答 → 不合格
  • B类：回顾时间跨度 < 5分钟 → 不合格
  • D类：信息在视频里已讲过 / 不需要搜索 → 不合格
  • D类：生成的题脱离视频实际场景 → 不合格
  • E类：触发事件超过10分钟 / 回答格式不一致 → 不合格
  • 所有类型：答案偏主观 / 指代不明确 → 不合格
